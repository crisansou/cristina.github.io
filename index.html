Sobre Mim

Atuei por muitos anos com análise e desenvolvimento de sistemas, utilizando tecnologias como PL/SQL, Forms e Reports. Durante minha trajetória também fui líder técnica e arquiteta de soluções, desempenhando papéis estratégicos em projetos de alta complexidade, como migração de sistema desenvolvido em Oracle Forms para Java.

No meu último cargo formal, atuei como consultora de serviços em nuvem no Azure, aprimorando minha expertise em tecnologias modernas e escaláveis.

Por ser detalhista e apaixonada por análise, direcionei minha carreira para a área de análise de dados. Hoje, desenvolvo habilidades em visualização de dados e storytelling, utilizando ferramentas como o Power BI para transformar informações complexas em insights.

Paixão pelo mercado financeiro: Nos meus momentos livres, sou trader de opções e utilizo a análise de dados para otimizar decisões e identificar oportunidades.

Em um mundo movido por dados, um analista é a ponte entre informações brutas e decisões estratégicas. Meu papel é ajudar empresas a compreender tendências, identificar oportunidades e mitigar riscos.

Fases do Projeto


Todo projeto será desenvolvido utilizando as fases a seguir:

1. Levantamento de Requisitos
2. Análise de Viabilidade
3. Definição das Tecnologias Necessárias
4. Definição do Cronograma
5. Início do Desenvolvimento
6. Reunião de Apresentação do Projeto
7. Validação do Cliente
8. Entrega após a Aprovação do Cliente

1. Levantamento de Requisitos:

  • Entender qual é o problema a ser resolvido
  • Quem serão as pessoas que terão acesso ao dashboard
  • Quais são as fontes de dados envolvidas e de quanto em quanto tempo estes dados são atualizados
  • Qual será a frequência da análise e qual é a granularidade desejada

2. Análise de Viabilidade:

  • Analisar se é possível acessar todas as bases necessárias
  • Avaliar se é possível extrair todos os KPI desejados

3. Definição das Tecnologias Necessárias:

  • Definir quais serão as ferramentas utilizada em cada etapa do projeto, por exemplo: SQL Server, Figma e Power BI

4. Definição do Cronograma:

  • Dividir o projeto em etapas
  • Definir datas para cada etapa e data da entrega final
  • Definir data para aceite do projeto ou ajustes caso necessário
  • Definir prazo de manutenção após aceite final/li>

5. Início do Desenvolvimento:

  • Desenvolver a etapa, tirar as dúvidas com o stakeholder e validar o que foi feito
  • Se tudo estiver de acordo começar o desenvolvimento da nova etapa e assim sucessivamente até a conclusão do projeto

6. Reunião de Apresentação do Projeto:

  • Apresentar tudo o que feito e disponibilizar dashboard para teste

7. Validação do Cliente:

  • Nesta fase o cliente irá testar todas as funcionalidades entregues, se estiver tudo ok ele deverá dar o aceite e caso contrário solicitar o ajuste

8. Entrega após a Aprovação do Cliente:

  • Implantar os artefatos e enviar a documentação do projeto
